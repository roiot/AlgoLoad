# PaperVizAgent-inspired Codex Skill (text-only)

Этот скилл переносит идеи из `google-research/papervizagent` в формат практических правил для Codex, **без установки и копирования кода** из внешнего репозитория.

## Что перенесено из идей PaperVizAgent
В основе — оркестрация ролей и замкнутый цикл улучшения:
1. **Retriever** — найти релевантные референсы/контекст.
2. **Planner** — построить план решения и структуру результата.
3. **Stylist** — привести результат к требуемому стилю и стандартам.
4. **Visualizer** — собрать финальный артефакт (код/док/ответ).
5. **Critic** — критически проверить и запустить итерацию улучшений.

## Когда применять
- Нужен системный подход к сложным задачам (много шагов, много ограничений).
- Важно качество финальной формы (читабельность, структура, стиль).
- Требуется итеративная шлифовка, а не одноразовый ответ.

## Ролевая схема для Codex

### 1) Retriever phase (Контекст)
- Собери только необходимые входы: требования пользователя, локальные инструкции, связанные файлы.
- Отдели **факты** от **предположений**.
- Зафиксируй пробелы в информации.

**Выход фазы:** краткий контекст-бриф (что известно / чего не хватает).

### 2) Planner phase (План)
- Сформулируй цель в проверяемом виде.
- Разбей на небольшие шаги с критериями «готово/не готово».
- Для каждого шага укажи артефакт проверки (команда, тест, дифф, пример).

**Выход фазы:** мини-план выполнения.

### 3) Stylist phase (Стандарты)
- Приведи решение к локальному стилю проекта.
- Убери лишнюю сложность и нецелевой scope.
- Проверь тон и формат финального ответа под задачу.

**Выход фазы:** стилистически согласованный черновик.

### 4) Visualizer phase (Сборка результата)
- Реализуй только необходимые изменения.
- Сохрани прозрачность: что изменено, где и почему.
- Подготовь воспроизводимые шаги проверки.

**Выход фазы:** готовый артефакт + проверки.

### 5) Critic phase (Итеративная критика)
Прогоняй минимум 2 цикла самооценки:
- Цикл A: корректность (решает ли цель полностью).
- Цикл B: минимализм (можно ли проще без потери качества).

Если обнаружены проблемы — вернись на нужную фазу и повтори.

## Шаблон итерации (быстрый)
1. Контекст (Retriever)
2. План (Planner)
3. Выравнивание стиля (Stylist)
4. Реализация (Visualizer)
5. Критика и правки (Critic)

## Чеклист качества перед финальным ответом
- [ ] Все требования пользователя покрыты явно.
- [ ] Нет выхода за scope.
- [ ] Есть проверка результата (команды/тесты/наблюдения).
- [ ] Есть краткий отчёт об изменениях.
- [ ] Есть заметки о рисках/ограничениях.

## Формат финального отчёта (рекомендуемый)
- **Сделано:** 3–6 пунктов.
- **Проверено:** конкретные команды и итоги.
- **Не делалось:** что сознательно оставлено вне scope.
- **Риски:** что может потребовать следующего шага.
